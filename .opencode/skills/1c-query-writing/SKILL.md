# Язык запросов 1С для аналитика

## Обязательно

- Запрос начинается с `ВЫБРАТЬ` (или `ВЫБРАТЬ ПЕРВЫЕ N`).
- Только read-only — никаких операторов изменения данных.
- Используй `ПЕРВЫЕ 50` / `ПЕРВЫЕ 100` если объём неизвестен.
- Фильтруй по `Номер`, `Дата`, `Контрагент`, `Ссылка` когда известны.
- Алиасы полей — латиница: `КАК OrderNumber`, `КАК OrderDate` (упрощает чтение JSON).

## Шаблон

```bsl
ВЫБРАТЬ ПЕРВЫЕ 20
    Doc.Номер КАК DocNumber,
    Doc.Дата КАК DocDate,
    Doc.Проведен КАК Posted
ИЗ
    Документ.ИмяДокумента КАК Doc
ГДЕ
    Doc.Номер = &Номер
УПОРЯДОЧИТЬ ПО
    Doc.Дата УБЫВ
```

Для MCP параметры `&Имя` не поддерживаются — подставляй значения в текст запроса с экранированием кавычек (`""` внутри строки).

## Антипatterns

- `ВЫБРАТЬ *` без фильтра по большим таблицам
- Выбор всех полей `Ссылка.*` — только нужные реквизиты
- Предположение имени объекта без проверки (ERP vs УТ vs БП)

## Ошибки платформы

- Если запрос упал — упрости: меньше полей, другие имена реквизитов, проверь `ПометкаУдаления`, `ЭтоГруппа` для справочников.
