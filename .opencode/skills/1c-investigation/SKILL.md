# Расследование ошибок 1С

Работает в **live** (с запросами к ИБ) и **offline** (без базы). Режим — skill `1c-work-modes`.

## Алгоритм (live — с базой)

**Порядок задаёт skill `1c-investigation-pipeline`:** connect → метаданные → **ИТС** → чек-лист для пользователя → при эскалации **код** (`onec_read_module`) → `onec_query` по согласию.

Ниже — справочник проверок в ИБ (для чек-листа или для `onec_query`, если пользователь согласен):

1. **Симптом** — что не так (не проводится, не отгружается, неверная сумма).
2. **Объект** — документ/элемент по номеру, дате, контрагенту.
3. **Шапка** — статус, проведение, организация, контрагент, сумма.
4. **Табличная часть** — номенклатура, количество, цена, резерв, серии.
5. **Связи** — основание, заказ, реализация, оплата.
6. **Движения** — регистры по документу.
7. **Типовые причины** — таблица ниже.
8. **Гипотеза** с уровнем уверенности.

## Алгоритм (offline — без базы)

1. **Симптом и контекст** — что сообщил пользователь; конфигурация и **версия** (обязательно спроси, если не указаны).
2. **Кейсы** — `onec_search_cases`.
3. **Типовые причины** — таблица ниже; помечай «вероятно», не «у вас точно».
4. **ИТС / форумы** — skill `1c-web-research`; форумы только как гипотезы.
5. **Ответ** — известные факты → гипотезы → **чек-лист проверок в ИБ**, когда доступ появится (конкретные объекты, регистры, запросы).

Не выдумывай значения реквизитов и остатков, которых не видел.

## Типовые причины (УТ/ERP)

| Симптом | Что проверить |
|---------|---------------|
| Заказ не отгружается | Статус заказа, процент отгрузки, связанные реализации, резервы, блокировка кредитного лимита |
| Документ не проводится | Заполненность реквизитов, дата запрета, права, отрицательные остатки |
| Неверная сумма | Цены, скидки, валюта, НДС, округление |
| Дубли / не тот контрагент | Поиск по ИНН, наименованию, несколько заказов с одним номером в разных периодах |

## Итерации

- Начинай с 1–2 узких запросов, не с «выбрать всё».
- Если данных недостаточно — следующий запрос по результатам предыдущего.
- Явно указывай, каких данных не хватает для окончательного вывода.
